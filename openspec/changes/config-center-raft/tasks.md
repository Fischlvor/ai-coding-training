# Tasks: 基于 Raft 的简化版配置管理中心

> 任务边界说明：每个任务只覆盖其标题所描述的最小闭环，不向外扩展到其他阶段的能力。
> 任务实现应遵循 Clean Architecture 分层，优先保证实体、用例、适配器、基础设施之间的边界清晰。

## Phase 1: 基础与底座

- [ ] Task 1.0: 确认 Docker Compose 部署骨架与网络命名
  - 边界：仅定义服务编排、网络名称、端口映射与配置挂载方式。
  - 不包含：应用业务逻辑、页面实现、Raft 业务适配。

- [ ] Task 1.1: 初始化项目目录与基础配置
  - 边界：仅完成项目目录、基础配置文件、运行入口和必要的依赖初始化。
  - 不包含：业务接口、数据模型、Raft 接入、前端页面。

- [ ] Task 1.2: 设计 PostgreSQL 表结构与约束索引
  - 边界：仅完成表结构、主键、唯一约束、外键、索引与字段类型定义。
  - 不包含：业务接口、Raft 接入、页面实现。

- [ ] Task 1.3: 定义配置中心的核心数据结构与状态机命令
  - 边界：仅定义 Entity / UseCase DTO / Command / Event 及其最小字段集合。
  - 不包含：HTTP 接口、前端页面、实际持久化实现、具体业务流程编排。

- [ ] Task 1.4: 定义模型映射与转换层
  - 边界：仅定义 Input / Output / DB / Query / Command / Event 之间的转换规则与映射器接口。
  - 不包含：具体业务 CRUD、数据库访问实现、Raft 提交逻辑。

- [ ] Task 1.5: 接入现有 Raft 课程工程作为一致性底座
  - 边界：仅完成 Raft 工程依赖接入、编译联通、基础启动与最小调用验证。
  - 不包含：配置业务逻辑、发布流程、灰度规则、SDK、后台页面。

- [ ] Task 1.6: 定义对外稳定的 Raft 调用接口
  - 边界：仅定义 GetState、Start、Kill、ApplyMsg 相关使用方式与封装层契约。
  - 不包含：Raft 内部实现细节、日志结构、选举算法调整。

- [ ] Task 1.7: 定义业务层与 Raft 层之间的命令适配器
  - 边界：仅完成业务请求到 Raft command 的封装，以及 ApplyMsg 到业务事件的转换入口与接口契约。
  - 不包含：具体业务 CRUD、发布逻辑、灰度规则、UI 页面。

- [ ] Task 1.8: 搭建 HTTP API 与后台页面的基础入口
  - 边界：仅建立服务路由、页面路由、基础页面骨架和健康检查入口。
  - 不包含：具体 CRUD 业务、发布逻辑、灰度逻辑、推送逻辑、后台完整功能。

## Phase 2: 领域核心

- [ ] Task 2.1: 实现分组维度的版本号生成与唯一性约束
  - 边界：仅实现 version_no 在 group 维度内的单调递增生成、唯一性检查与事务内分配。
  - 不包含：发布逻辑、灰度逻辑、Raft 复制细节。

- [ ] Task 2.2: 实现应用管理的增删改查
  - 边界：仅实现应用实体的增删改查与基础校验。
  - 不包含：环境、分组、配置项、版本、发布、Raft、后台页面。

- [ ] Task 2.3: 实现环境管理的增删改查
  - 边界：仅实现环境实体的增删改查与基础校验。
  - 不包含：应用、分组、配置项、版本、发布、Raft、后台页面。

- [ ] Task 2.4: 实现配置分组管理与配置项管理
  - 边界：仅实现分组与配置项的增删改查、命名唯一性校验、归属关系校验。
  - 不包含：版本历史、发布、回滚、灰度、推送、SDK、后台完整联动。

- [ ] Task 2.5: 实现草稿版本、版本历史与发布记录
  - 边界：仅实现草稿版本保存、版本列表查询、发布记录生成与状态维护。
  - 不包含：灰度命中、推送通知、SDK 缓存刷新、Raft 提交细节。

- [ ] Task 2.6: 实现发布与回滚流程
  - 边界：仅实现发布前校验、发布生效、回滚生效、重复发布/重复回滚幂等处理。
  - 不包含：灰度策略计算、推送协议、SDK 解析、后台复杂审批流。

- [ ] Task 2.7: 实现灰度发布规则的创建、匹配与记录
  - 边界：仅支持比例、标签、白名单三种灰度规则及命中记录（GrayRule / GrayRecord）。
  - 不包含：复杂规则引擎、动态策略编排、多维度优先级系统、监控平台接入。

- [ ] Task 2.8: 实现实时配置推送事件生成与订阅管理
  - 边界：仅实现标准化事件生成、订阅登记、推送发送、断线补偿拉取入口（Subscription / ChangeEvent）。
  - 不包含：完整消息队列系统、复杂流式协议、Prometheus、图表展示。

- [ ] Task 2.9: 实现 Go SDK 的拉取、订阅与缓存刷新能力
  - 边界：仅实现 Go SDK 的初始化、拉取配置、订阅变更、缓存刷新、灰度结果判断（ConfigSnapshot）。
  - 不包含：其他语言 SDK、复杂插件机制、后台管理页面逻辑。

- [ ] Task 2.10: 实现可视化管理后台的配置列表、发布、回滚与集群状态页面
  - 边界：仅实现核心后台页面与基础交互，覆盖配置查询、发布、回滚、集群状态展示。
  - 不包含：审批流、角色权限体系、复杂报表、运营大盘、Prometheus 图表。

## Phase 3: 一致性与恢复

- [ ] Task 3.1: 将配置写请求封装为 Raft command 并提交
  - 边界：仅处理配置相关写请求的 command 封装、提交与结果返回。
  - 不包含：业务模型重构、后台页面、SDK 逻辑、灰度规则计算。

- [ ] Task 3.2: 实现状态机对已提交 command 的应用
  - 边界：仅实现已提交 command 到配置元数据、版本和发布记录的状态更新。
  - 不包含：网络传输、前端展示、SDK 缓存刷新、复杂持久化策略。

- [ ] Task 3.3: 实现业务层与 Raft 层之间的命令适配器
  - 边界：仅完成业务请求到 Raft command 的封装，以及 ApplyMsg 到业务事件的转换入口落地。
  - 不包含：具体业务 CRUD、发布逻辑、灰度规则、UI 页面。

- [ ] Task 3.4: 处理 leader 不可用与多数派未形成的写入失败场景
  - 边界：仅处理写请求失败、重试提示、拒绝确认成功、保持当前生效版本不变。
  - 不包含：完整故障转移编排、跨机房容灾、人工干预流程。

- [ ] Task 3.5: 实现节点重启后的日志恢复与状态恢复
  - 边界：仅完成 Raft 日志恢复、基础状态恢复、恢复期间禁止写入。
  - 不包含：数据中心级恢复、复杂快照系统、备份编排。

- [ ] Task 3.6: 处理重复事件投递与客户端断线补偿拉取
  - 边界：仅实现事件幂等识别、重复投递不改变最终状态、断线后补偿拉取。
  - 不包含：完整离线消息系统、跨区域消息同步、推送监控平台。

- [ ] Task 3.7: 实现 Raft ApplyMsg 驱动的状态机应用入口
  - 边界：仅处理已提交命令的分发、幂等判定与状态机调用。
  - 不包含：HTTP 接口、前端页面、灰度策略、SDK 缓存刷新细节。

## Phase 4: 测试与优化

- [ ] Task 4.1: 编写域管理与版本发布的单元测试
  - 边界：仅覆盖应用、环境、分组、配置项、版本、发布和回滚的核心单测。
  - 不包含：全量 UI 测试、压力测试、Grafana/Prometheus 验证。

- [ ] Task 4.2: 编写灰度发布与推送流程的集成测试
  - 边界：仅覆盖灰度命中、推送事件、补偿拉取、重复事件幂等行为。
  - 不包含：大规模压测、复杂网络故障注入、服务发现测试。

- [ ] Task 4.3: 编写 Raft 一致性与恢复场景测试
  - 边界：仅覆盖 leader 切换、多数派未形成、提交失败不部分生效、恢复流程。
  - 不包含：跨机房一致性、复杂共识协议扩展、分布式事务测试。

- [ ] Task 4.4: 完成 Go SDK 基本使用示例与联调验证
  - 边界：仅提供 Go SDK 的最小使用示例、拉取、订阅、缓存刷新联调验证。
  - 不包含：多语言示例、SDK 插件体系、企业级接入文档。

- [ ] Task 4.5: 完成管理后台演示页面与核心流程验证
  - 边界：仅验证后台的查询、发布、回滚、集群状态展示和错误提示。
  - 不包含：复杂权限、审批流、报表中心、监控大盘。
